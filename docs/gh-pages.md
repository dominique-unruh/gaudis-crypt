# GitHub Pages (Local Build)

This repo publishes a GitHub Pages site from the `gh-pages` branch.

Manual site sources live on `main` under `doc/website/`.
Autogenerated Lean API docs are built locally using `doc-gen4` (as a dependency of the main Lake project).

## One-time setup

1. Make sure you have Lean installed (via `elan`) and a working toolchain for this repo.
2. Fetch dependencies and (optionally) mathlib cache:

   - `lake update`
   - `lake exe cache get`  (optional but recommended)

## Build + publish

Run:

`python3 scripts/build-gh-pages.py`

This will:

1. Run `lake update`, `lake exe cache get` (best-effort), and then `lake build DocEntrypoint:docs`.
2. Combine:
   - `site/` (manual pages) into the root of `gh-pages`
   - doc-gen4 output (if any) into `gh-pages/lean/`
3. Commit the result on `gh-pages`.

To also push to GitHub:

`python3 scripts/build-gh-pages.py --push`

## Failure / stale behavior

If the doc build fails:

1. The manual site content is still updated.
2. The existing `gh-pages/lean/` content is kept as fallback (stale), and any newly generated doc output files are overlaid.
3. A prominent warning banner is injected into `index.html` and `lean/index.html`.
4. Build info is written to `build-status.txt` and `build-status.json` on `gh-pages`.

If the doc build succeeds:

1. `gh-pages` is rebuilt from scratch (except optional `CNAME` if you used `--keep-cname`).
